### 번역 적용 페이지 상세

#### page.tsx (홈페이지)
- Hero 섹션: 타이틀, 서브타이틀
- 검색창: placeholder, 키워드 결과 메시지
- 업종 버튼: 번역 키로 업종명 표시
- 최근 작업 섹션: 타이틀
- Footer 정보: 크레딧/4K/상업적 사용

#### pricing/page.tsx (가격 페이지)
- 헤더: 타이틀, 서브타이틀
- 탭: 크레딧 패키지, 구독 플랜
- 패키지 카드: 인기 뱃지, 크레딧, 보너스, 구매 버튼
- 플랜 카드: 추천 뱃지, 월간 크레딧, 구독/현재 플랜 버튼
- FAQ 섹션: 4개 질문/답변

#### GalleryClient.tsx (갤러리)
- 헤더: 타이틀, 총 이미지 수
- 검색/정렬: placeholder, 정렬 옵션
- 빈 상태: 타이틀, 설명, 버튼
- 에러 상태: 메시지, 재시도 버튼
- 페이지네이션: 이전/다음
- 이미지 카드: 드롭다운 메뉴 (다운로드, 삭제)
- 라이트박스: 타이틀, 프롬프트 라벨, 버튼
- 삭제 확인: 타이틀, 설명, 버튼

#### result/page.tsx (결과 페이지)
- 로딩 상태: 메시지
- 결과 없음: 메시지, 홈 버튼
- 에러 상태: 타이틀, 돌아가기/다시 시도 버튼
- 헤더: 타이틀, 이미지 수/크레딧 사용량, 액션 버튼
- 생성 정보: 프롬프트, 네거티브 프롬프트, Provider/Model/시간
- 유사 워크플로우: 타이틀
- 라이트박스: 타이틀, 저장/다운로드/공유 버튼
- Alert 메시지: 저장 성공/실패

## 테스트 방법
1. `npm run dev` 실행
2. `/` 접속 → 홈페이지 한국어 텍스트 확인
3. `/en` 접속 → 홈페이지 영어 텍스트 확인
4. `/pricing` → 가격 페이지 번역 확인
5. `/gallery` → 갤러리 페이지 번역 확인
6. 이미지 생성 → `/result` 결과 페이지 번역 확인
7. 언어 토글로 한/영 전환 테스트

## 기술 참고

### 번역 사용 패턴
```typescript
// 단일 namespace
const t = useTranslations("pages.home");
return <h1>{t("title")}</h1>;

// 다중 namespace
const t = useTranslations("pages.home");
const tWorkflow = useTranslations("workflow.industries");
return <span>{tWorkflow(`${industry.id}.name`)}</span>;

// 파라미터 사용
t("totalImages", { count: 42 })  // "총 42개의 이미지"
t("bonus", { bonus: 10 })        // "+10% 보너스"

// props로 번역 함수 전달
<GalleryImageCard t={t} />
```
