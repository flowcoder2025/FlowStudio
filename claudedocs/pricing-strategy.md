## 💰 1. 크레딧 시스템

### 기본 단위
- **1 크레딧 = ₩100**
- 환율 기준: $1 = ₩1,400

### 이미지 생성 크레딧 소비

| 작업 | 크레딧 | 가격 | 생성 수량 | API 비용 | Gross Margin |
|------|--------|------|-----------|----------|--------------|
| **2K 생성 1회** | 20 | ₩2,000 | 4장 | $0.536 (₩750) | 62.5% |
| **업스케일 1회** (2K→4K) | 10 | ₩1,000 | 1장 | $0.24 (₩336) | 66.4% |

**장당 비용 (사용자 입장)**:
- 2K 생성: ₩2,000 ÷ 4장 = **₩500/장**
- 4K 획득: ₩2,000 (생성) + ₩1,000 (업스케일) = **₩3,000/장**
  - 또는 4장 중 1장만 업스케일 시: ₩500 + ₩1,000 = **₩1,500/장**

**선정 근거**:
- SaaS 산업 표준 60-70% 마진 달성 ✅
- 심리적 가격대 (20 크레딧 = 2,000원 단위) ✅
- 4장 묶음 제공으로 사용자 체험 가치 극대화 ✅
- 업스케일링을 프리미엄 옵션으로 포지셔닝 ✅
- API 가격 변동 대응 여지 확보 ✅

**경쟁사 비교**:
- Midjourney: 월 $10에 약 200장 = 장당 ₩70
- DALL-E 3: 장당 $0.04 = ₩56
- FlowStudio 2K: 장당 ₩500 (7배 비싸지만 한국 특화 + 이커머스 최적화)

## ⚠️ 6. 위험 관리

### 주요 위험 요소 및 대응 전략

**1. API 가격 변동 위험** (High Impact)
- **위험**: Google이 Gemini API 가격 인상 시 마진 급격히 축소
- **대응**:
  - 크레딧 가격 조정 조항 명시 (이용약관)
  - "API 비용 변동 시 30일 전 공지 후 가격 조정 가능"
  - 10% 버퍼 고려 시: 2K → 22 크레딧, 4K → 44 크레딧
  - 정기적인 경쟁사 가격 모니터링

**2. 무료 크레딧 남용** (Medium Impact)
- **위험**: 멀티 계정 생성으로 무한 무료 크레딧
- **영향**: 무료 크레딧 2배 증액으로 위험 증가
- **대응**:
  - 이메일 인증 필수
  - IP/디바이스 핑거프린팅 강화
  - 사업자 회원은 사업자번호 중복 검증
  - 레퍼럴은 첫 유료 충전 후 지급
  - 의심 계정 자동 탐지 시스템 (AI 기반)
  - 일일 생성 제한 (Free: 1회/일 = 4장)

**3. 전환율 저조** (Medium Impact)
- **위험**: Free → Paid 전환율 5% 미만
- **산업 평균**: 2-5% (SaaS)
- **대응**:
  - 온보딩 개선 (첫 이미지 생성까지 3분 이내)
  - 이메일 마케팅 자동화 (무료 크레딧 소진 3일 전 알림)
  - 제한적 기능으로 유료 전환 유도 (워터마크, 동시 생성 제한)
  - A/B 테스트를 통한 가격 최적화
  - 4장 묶음 제공으로 제품 가치 체험 강화 ✅

**4. 스토리지 비용 폭증** (Low Impact)
- **위험**: Business 무제한 플랜 사용자가 TB급 사용
- **대응**:
  - Fair Use Policy 적용 (월 1TB 제한)
  - 이미지 자동 압축 (품질 손실 최소화, WebP 전환)
  - 90일 미사용 이미지 자동 삭제 안내
  - 과도 사용 시 개별 협의

**5. 4장 묶음 생성의 UX 리스크** (Medium Impact)
- **위험**: 사용자가 1장만 필요할 때도 4장 생성 → 크레딧 낭비 인식
- **대응**:
  - "4장을 함께 생성하여 최적의 결과 선택" 메시지
  - 생성 후 즉시 선택/삭제 기능 제공
  - 업스케일링 1장 옵션 제공 (10 크레딧)
  - 다양한 스타일 변형 자동 제공으로 가치 강조

- DB 스키마: `Credit` 모델 추가
  ```prisma
  model Credit {

    userId    String   @unique
    balance   Int      @default(0)
    createdAt DateTime @default(now())
    updatedAt DateTime @updatedAt

  model CreditTransaction {
  ```

## 🎯 결론

### 핵심 권장사항
1. **크레딧 가격**: 2K 생성 1회 (4장) → 20 크레딧, 업스케일 (2K→4K) 1회 → 10 크레딧 (확정)
2. **생성 방식**: 2K 생성만 제공, 4K는 업스케일링 전용
3. **구독 분리**: 크레딧과 구독 완전 분리 (한국 정책 준수)
4. **손익분기점**: 런칭 1개월 내 달성 가능 (현실적)
5. **마케팅 투자**: 초기 ₩4,500,000 (합리적 CAC)
6. **업스케일 플로우**: 2K 생성 (4장) → 선택 → 업스케일 (1장) → 4K 완성

### 2K 생성 + 업스케일링 구조의 장점
- ✅ **간단한 가격 구조**: 2K 생성과 업스케일 2가지만
- ✅ **사용자 만족도 향상**: 4장 중 최적 선택 후 업스케일
- ✅ **체험 가치 증가**: 무료 크레딧 30으로 1회 생성 + 1회 업스케일 체험
- ✅ **전환율 향상 기대**: 생성과 업스케일 플로우 체험 → 구매 유도
- ✅ **업스케일 프리미엄화**: 4K는 선택적 업그레이드로 포지셔닝
- ✅ **차별화 요소**: 경쟁사 대비 독특한 UX

### 주의사항
- ⚠️ **무료 크레딧 남용**: 방지 시스템 필수 (IP 핑거프린팅, 일일 제한)
- ⚠️ **업스케일 활용도**: 목표 30% 달성 필요 (낮으면 수익 감소)
- ⚠️ **UX 메시지 중요**: "4장 생성 → 베스트 선택 → 업스케일" 플로우 명확히 안내
